# Задание 4b. NoSQL (MongoDB)

## 1. Хранение данных

Коллекции пользователей, отелей, туров и бронирований  можно хранить отдельно и связывать через references, так как они используются часто и могут изменяться независимо друг от друга. В embedded лучше хранить только небольшие данные, которые относятся только к конкретному бронированию, например контактные данные гостя или информацию об оплате.

## 2. Индексы

userId: для быстрого поиска всех бронирований конкретного пользователя;

roomId: для проверки занятости номера;

checkIn + checkOut (составной индекс):  для поиска бронирований по диапазону дат;

status: для фильтрации по статусу бронирования;

roomId + checkIn + checkOut (составной индекс):   для эффективной проверки пересечения дат и предотвращения двойного бронирования номера.

## 3. Тестирование данных

При тестировании в MongoDB Compass можно использовать фильтры, Schema Analysis и просмотр документов для проверки наличия обязательных полей и корректности их типов данных.

В mongo shell можно выполнять запросы для поиска некорректных записей:
db.bookings.find({ userId: { $exists: false } }) 
db.bookings.find({ $expr: { $gt: ["$checkIn", "$checkOut"] } })
